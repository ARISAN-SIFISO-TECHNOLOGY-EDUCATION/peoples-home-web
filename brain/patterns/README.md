## How to add a pattern

When a design or architecture decision recurs across two or more apps, it is a pattern.
Add a new file named `<pattern-slug>.md` and list it in this README.

A pattern file should cover:
1. **What it is** (one paragraph)
2. **Why we use it** (the problem it solves)
3. **How it is implemented** (the technical shape)
4. **Where it is used** (app list with notes)
5. **Rules and constraints** (what you must never do with it)
6. **Evolution** (how it has changed over time — append, never overwrite)
