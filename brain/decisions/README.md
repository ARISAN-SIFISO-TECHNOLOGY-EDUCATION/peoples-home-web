# Decisions

> The reasoning archive for The People's Home ecosystem.
> Every significant structural decision — the ones that would be confusing to reverse-engineer
> from the code alone — gets an entry here.
>
> **Protocol:** Append only. Never edit existing entries. Add the date.
> The full decision log is in [`DECISIONS.md`](./DECISIONS.md).

---

## What counts as a decision

Record here when you:
- Chose between two real alternatives (e.g., Journey vs. Chapter structure)
- Made a non-obvious constraint (e.g., "no Google Play after June 2026")
- Rejected an approach and why (e.g., "Atelier chassis rejected for Micro Founders")
- Changed a previous direction (e.g., web-first pivot on 2026-06-12)

Do NOT record:
- Implementation details (those belong in code comments or app memory files)
- Ongoing tasks (those belong in SESSION_HANDOFF.md or ROADMAP.md)
