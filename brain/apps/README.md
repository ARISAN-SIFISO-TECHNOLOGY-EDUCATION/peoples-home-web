# Application Memories

> One memory per **World-A learning app**. Each captures the *evolution of ideas*: why the app
> exists, the original vision, major design iterations, architectural decisions, rejected
> alternatives, why the current approach was chosen, and future direction.
>
> For **live** facts (repo, URL, build config, current status) see the operational
> [`../../ECOSYSTEM.md`](../../ECOSYSTEM.md) and [`../../ROADMAP.md`](../../ROADMAP.md).

---

## Template (every app memory follows this)

1. **What it is** — one paragraph.
2. **Pillar & audience.**
3. **Why it exists** — the gap it fills.
4. **Original vision.**
5. **Design iterations / how the thinking evolved.**
6. **Architectural decisions** (and **rejected alternatives**).
7. **Current state** (links to operational docs).
8. **Future direction.**
9. **Related.**
